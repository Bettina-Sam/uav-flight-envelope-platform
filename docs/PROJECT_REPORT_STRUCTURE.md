# Project Report Structure

Suggested chapter structure for your written DRDO internship report. Map each section
to material already produced in this project's docs.

1. **Title Page & Certificate** (per your institute's format)

2. **Abstract** — 150-200 words: physics-informed ML platform, predicts UAV flight
   envelope & altitude optimization, synthetic-data surrogate model, web deployment.

3. **Introduction**
   - Motivation: why UAV flight envelope prediction matters for mission planning
   - Problem statement: predict min/max/recommended altitude + full performance envelope
   - Objectives (list the 11 predicted outputs from `README.md`)

4. **Literature Review / Background**
   - Aircraft performance theory (cite standard aerospace performance texts, not this
     document, for the underlying theory — see the equations in `FORMULA_SHEET.md`)
   - Physics-informed ML / surrogate modeling as a methodology
   - Why real UAV flight-test data is restricted for this class of project

5. **System Design & Methodology**
   - Architecture diagram (frontend/backend/ML split — draw from `README.md` folder tree)
   - Physics engine design (`FORMULA_SHEET.md`)
   - Synthetic dataset generation (`DATASET_DESCRIPTION.md`)
   - ML pipeline: models compared, selection criteria, metrics (`ML_METHODOLOGY.md`)
   - Explainable AI approach (permutation + native feature importance)

6. **Implementation**
   - Backend: FastAPI, endpoints, request/response flow
   - Frontend: React/TypeScript/Tailwind, page-by-page walkthrough (`USER_MANUAL.md`)
   - PWA implementation (manifest, service worker, install flow)

7. **Results**
   - Model comparison table (copy from `ML_METHODOLOGY.md`)
   - Feature importance findings — which parameters matter most for altitude
   - Example predictions: pick 2-3 UAV configurations, show physics vs ML side by side
   - Screenshots of the dashboard, gauges, sensitivity analysis

8. **Discussion**
   - Why recommended altitude ≠ average (this is a strong point to emphasize — it shows
     engineering judgment, not a lookup-table shortcut)
   - Where the ML model agrees/disagrees with physics, and why (e.g. recommended
     altitude has lower R² — explain the argmax-of-flat-function reasoning)

9. **Limitations**
   - Synthetic data only, not validated against real flight-test telemetry
   - No wind/gust/turbulence modeling
   - Empirical (not measured) propeller efficiency curve
   - Directly copy the "Synthetic Assumptions" list from the app's About page

10. **Future Work**
    - SHAP explanations, partial dependence plots
    - CatBoost addition, real-data validation, wind modeling

11. **Conclusion**

12. **References**
    - Standard aerospace performance textbooks (e.g. Anderson's Aircraft Performance
      and Design) for the underlying equation theory
    - scikit-learn, XGBoost, FastAPI, React documentation for tooling citations

13. **Appendices**
    - Full formula sheet (`FORMULA_SHEET.md`)
    - Full parameter table (`DATASET_DESCRIPTION.md`)
    - Sample PDF report generated by the platform (Report Generation page)
    - Source code structure listing
